## 🏛️ Swarm Intelligence & Governance (v12.5)

DevForge is now powered by a **Ruflo-inspired swarm architecture**, moving beyond single-agent interactions into a coordinated "Nervous System" for repository management.

### 🧠 1. Neural Intelligence Layer (`NeuralMemory`)
Persistent agent intelligence mimicking **RuVector**. It stores "Winning Patterns" and "Failure Modes" across sessions, allowing the swarm to learn from every successful mutation and build failure.

### 🗳️ 2. Swarm Consensus Engine (`SwarmConsensus`)
High-risk decisions (mutations, migrations, pivots) now require a **Swarm Vote**. This Byzantine-style fault tolerance ensures that architectural changes are vetted by multiple specialized agents (CEO, CTO, Architect) before execution.

### 🌊 3. Blue Ocean Market Discovery (`MarketStrategist`)
The system doesn't just react to trends—it creates markets. The **Market Strategist** identifies underserved technical gaps and plans "Product Alpha" launches to achieve maximum ecosystem dominance.

### 🧹 4. Autonomous Reliability (`TechDebt` & `Debug` Agents)
- **Debt Liquidation**: Automatically scans for code smells and legacy patterns.
- **Self-Healing**: Autonomous investigation and repair of failed CI/CD builds.
- **Stack Migration**: Plans and executes structural upgrades from legacy frameworks.
- **Generative Fraud Detection**: A RAG-powered Fraud Analysis Agent that evaluates high-risk events by cross-referencing live transactional heuristics against historical vector databases, making autonomous security interventions.
- **T-Pot Honeypot Network**: The Threat Intel Agent deploys low-interaction mock network listeners (SSH, HTTP, Telnet) exactly like T-Pot, captures live attack payloads, and uses LLM intelligence to profile threat actors and generate defensive signatures automatically.

## 🛠️ Getting Started

### 📦 Prerequisites
- **OpenAI API Key**: Powering the swarm intelligence.
- **GitHub Token**: For autonomous repo creation and PR management.
- **Docker & Docker Compose**: For the full production stack.

### 🛠️ One-Click Deployment
Run the full production swarm using Docker:

```bash
docker-compose up --build
```

Access the **Mission Control** dashboard at: `http://localhost:5180`
Access the **API Swagger** docs at: `http://localhost:8000/docs`
